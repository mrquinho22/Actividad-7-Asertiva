# 1. Definición del problema

En el ámbito académico, muchos estudiantes tienen dificultades para organizar sus tareas, trabajos y fechas de entrega. Esto puede provocar retrasos en las entregas, estrés académico y una mala gestión del tiempo.

Debido a esta situación, se propone el desarrollo de un **Sistema de Gestión de Tareas Académicas**, cuyo objetivo es permitir que los usuarios registren sus actividades académicas, asignen prioridades, den seguimiento al progreso de sus tareas y mantengan una mejor organización de sus responsabilidades.

Este sistema busca facilitar la planificación del trabajo académico y mejorar la productividad de los estudiantes mediante una herramienta sencilla de seguimiento de actividades.

---

# 2. Funcionalidades principales

## Registro de tareas académicas
El usuario podrá agregar nuevas tareas ingresando información como el nombre de la actividad, descripción, curso y fecha de entrega.

## Asignación de prioridad
Las tareas podrán clasificarse según su nivel de prioridad:
- Alta
- Media
- Baja

Esto permitirá identificar qué actividades deben realizarse primero.

## Actualización del estado de tareas
Cada tarea podrá cambiar su estado a medida que avanza:
- Pendiente
- En proceso
- Completada

## Visualización del progreso
El sistema mostrará el progreso del usuario indicando cuántas tareas han sido completadas y cuáles siguen pendientes.

## Organización por curso o fecha
Las tareas podrán filtrarse por materia o por fecha de entrega para facilitar la planificación académica.

---

# 3. Roles del sistema

## Estudiante
Es el usuario principal del sistema. Puede:
- Registrar tareas
- Asignar prioridades
- Actualizar el estado de las tareas
- Consultar su progreso académico

## Docente
Puede visualizar las tareas asignadas y el seguimiento del progreso académico de los estudiantes.

## Administrador
Se encarga del mantenimiento del sistema, la gestión de usuarios y el correcto funcionamiento de la plataforma.

---

# 4. Flujo de uso del sistema

1. El usuario inicia sesión en la plataforma.
2. Registra una nueva tarea académica.
3. Asigna una prioridad y una fecha de entrega.
4. El sistema guarda la tarea en la lista correspondiente.
5. El usuario consulta las tareas pendientes.
6. A medida que avanza, actualiza el estado de la tarea.
7. El sistema actualiza el progreso general del usuario.

---

# 5. Evento simulado: pérdida de conexión

Durante el desarrollo del mini-proyecto se simuló la situación en la que un integrante del equipo perdió la conexión a internet.

Para resolver esta situación se aplicó una estrategia de **redistribución de tareas**, donde los miembros restantes asumieron las actividades pendientes con el objetivo de continuar el trabajo sin afectar el tiempo establecido.

Esta decisión permitió mantener la continuidad del proyecto y finalizar correctamente el documento final.
